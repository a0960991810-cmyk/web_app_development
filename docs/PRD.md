# 產品需求文件 (PRD) - 工作管理系統

## 1. 專案概述

- **背景與動機**：現代人生活忙碌，需要有效的方式來追蹤和管理日常代辦事項。一個簡單好用的工作管理系統能協助使用者組織工作，減少遺忘，提升生活與工作效率。
- **目標用戶**：需要管理日常工作的個人用戶（如學生、上班族等）。
- **核心價值主張**：提供一個直覺、輕量且易於使用的介面，讓使用者能快速地管理與追蹤工作事項，並透過行事曆和狀態篩選功能，清楚掌握每個任務的進度。

## 2. 功能需求

以下列出系統的主要功能需求與對應的使用者故事：

1. **新增工作資訊**
   - 作為 **個人用戶**，我希望 **能夠新增一筆工作項目（包含標題、預期完成日期等）**，以便 **記錄我需要完成的任務**。
2. **刪除工作資訊**
   - 作為 **個人用戶**，我希望 **能夠刪除不再需要或輸入錯誤的工作項目**，以便 **保持工作清單的整潔**。
3. **更改工作資訊**
   - 作為 **個人用戶**，我希望 **能夠編輯已存在的工作內容（如重設截止日或修改任務名稱）**，以便 **隨時反映任務的最新狀況**。
4. **標記工作完成**
   - 作為 **個人用戶**，我希望 **能夠將已做完的任務標記為「完成」**，以便 **清楚區分出哪些事情還要處理，並獲得成就感**。
5. **顯示所有工作清單**
   - 作為 **個人用戶**，我希望 **能夠在一個畫面上看到我所有的工作項目**，以便 **快速瀏覽和掌握整體工作清單**。
6. **行事曆**
   - 作為 **個人用戶**，我希望 **能以行事曆的方式檢視我的工作安排**，以便 **清楚了解每天的工作量與即將到期的任務**。
7. **依完成狀態篩選**
   - 作為 **個人用戶**，我希望 **能夠篩選只顯示「已完成」或「未完成」的工作**，以便 **專注在當前需要被執行的任務上**。

## 3. 非功能需求

- **技術限制**：系統需使用 Python Web 框架 Flask 搭配模板引擎 Jinja2，以及輕型關聯式資料庫 SQLite 進行實作。
- **效能考量**：
  - 需確保在個人資料量（幾千筆以內）的情況下，查詢清單與切換行事曆的載入速度迅速順暢（建議在1秒內）。
  - 介面需要有良好的響應式設計（RWD），在電腦螢幕或手機上皆可順暢操作。
- **安全考量**：需防範常見的網頁攻擊，如需防範 SQL Injection 以及基本的跨站腳本攻擊 (XSS)，避免個人工作資訊被竊取或是意外損失。

## 4. MVP 範圍

為能快速驗證並建構出好用的產品，我們將開發階段分級如下：

- **Must Have（必須具備，系統最核心的功能）**
  - 新增工作資訊
  - 顯示所有工作清單
  - 標記工作完成
  - 刪除工作資訊
- **Should Have（應該具備，有了會更好用的重要功能）**
  - 更改工作資訊
  - 依完成狀態篩選（顯示全部、已完成、未完成）
- **Nice to Have（最好具備，進階優化體驗）**
  - 行事曆視圖（以月曆介面顯示每日待辦事項）
